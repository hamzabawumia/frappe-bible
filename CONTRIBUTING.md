# Contributing

Thanks for helping improve Frappe Bible. Please follow these guidelines:
- Open an issue for major changes or chapter proposals.
- Add chapters as separate Markdown files under docs/ and submit a PR.
- Keep chapters focused: Objectives, Try it, Challenge, Summary.
- Ensure any example code is MIT-licensed and has a short README in examples/.

Editorial checks
- Build mkdocs locally to ensure no broken links: mkdocs build
- Run markdown lint (optional)

Style
- Use US English
- Code blocks must be copy-pastable
